
## 🎮 基于 Go 的多人联机游戏服务器与 Unity 客户端

### 🛠 技术栈
`Go` · `C#` · `Unity3D` · `TCP` · `gRPC` · `Redis` · `MongoDB` · `WebSocket` · `RabbitMQ` · `Docker`

### 🌟 核心系统
实现在线查询系统、玩家系统、房间系统、好友系统、简易商城系统、排行榜系统。

### 1️⃣ 网络协议与高并发设计
*   **自定义协议**：基于 TCP 长连接自定义帧协议（长度头 + JSON），有效处理 **粘包/半包** 问题，按业务逻辑实现消息分离。
*   **并发安全**：服务端采用 `Channel` + `RunLoop` + `RWMutex` 架构，保障高并发下的数据一致性。
*   **性能压测**：单机支持 **5000+ 并发长连接**，吞吐量达 **26000+ 包/秒**，丢包率小于 0.01%。
*   **客户端处理**：采用 `BeginReceive` 异步循环拆包，网络线程入队、主线程批量分发，确保联机交互的高频率与稳定性。

### 2️⃣ 战斗同步与状态管理
*   **同步机制**：采用 **服务端权威 + 状态同步** 模式，判定技能 CD、超距命中及合法移动。
*   **广播优化**：服务端以 **30Hz** 频率广播场景状态，引入 **AOI 裁剪** 技术减少无效广播开销。
*   **抗抖动设计**：实现基于 `Seq` 序号的动作包去重与乱序检测，引入 **快速重传机制** 降低网络丢包影响。
*   **表现优化**：客户端通过 `Lerp` 插值优化移动平滑度；支持断线后自动从服务端/持久化数据恢复房间等联机上下文。

### 3️⃣ 微服务、异步解耦与持久化
*   **微服务架构**：基于 `gRPC` 拆分排行榜与商城服务，排行榜利用 `Redis ZSet` 实现实时排序。
*   **数据一致性**：商城购买链路引入 `req_id` 幂等、唯一索引与事务；针对跨服务结算，采用 **状态表模式** 配合 `MongoDB` 事务保障钱包余额的原子性。
*   **消息解耦**：聊天/私聊业务投递至 `RabbitMQ` 异步写入 `MongoDB`，主链路与落库逻辑解耦，通过 **手动 ACK** 保障消息不丢失。

### 4️⃣ 分布式架构、缓存与链路追踪
*   **网关转发**：`Gateway` 作为统一入口，配合 `Redis` 实现跨节点路由转发。
*   **缓存策略**：利用 `Redis` 实现接口幂等去重、聊天限流、分布式锁（改名逻辑）及防穿透的玩家数据缓存。
*   **排障溯源**：设计全局日志体系，按时间、用户、`req_id` 建立复合索引，实现快速的问题排查与溯源。

### 5️⃣ 内容安全与部署
*   **内容审计**：服务端集成 **DFA + Trie 树** 实现敏感词过滤与标签清洗。
*   **容器化运维**：支持环境变量注入，采用 **Docker** 容器化部署，提升多环境（开发/测试/生产）发布的一致性与可维护性。
