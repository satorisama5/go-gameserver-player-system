基于 Go 的多人联机游戏服务器与 Unity 客户端   
技术栈：Go, C#, Unity3D, TCP, gRPC, Redis, MongoDB, WebSocket，Rabbitmq
实现在线查询系统，玩家系统，房间系统，好友系统，简易商城系统，排行榜系统
1. 网络协议与高并发设计：基于 TCP 长连接自定义帧 [长度头+JSON] 协议处理粘包/半包，按业务内容将消息分离；服务端以Channel+RunLoop+RWMutex保障并发安全。单机压测验证5000并发长连接，吞吐量 26000+ 包/秒，丢包率 < 0.01%。客户端采用BeginReceive异步循环拆包，网络线程入队、主线程批量分发，支撑高频联机交互。
2.战斗同步与状态管理:实现基于 Seq 序号的动作包去重与乱序检测，引入简单快速重传机制降低丢包影响；服务器权威+状态同步判定技能CD与超距命中与合法移动；服务端以 30Hz 广播场景状态，配合 AOI 裁剪减少无效广播，客户端 Lerp 插值优化移动平滑度；支持自动重连后从服务端/持久化数据恢复位置与房间等联机上下文。
3.微服务、异步解耦与持久化： 基于 gRPC 拆分排行榜与简易商城服务，排行榜使用 Redis ZSet 实时排序；商城购买链路引入 req_id 幂等、唯一索引与事务保证并发一致性，针对跨服务资产结算场景采用状态表模式，利用MongoDB 事务保障钱包余额的原子性，未完成的Redis榜分通过后台Worker轮询重试。聊天/私聊等业务投递 RabbitMQ 异步写入 MongoDB，主链路收发与落库解耦，消费者手动 ACK 保障消息不丢失。
4分布式架构、缓存与链路追踪：Gateway对外暴露统一入口，配合Redis实现简单跨节点路由转发； Redis 实现接口幂等去重、聊天限流、私聊单调 Seq、改名分布式锁，以及防穿透的玩家数据缓存；设计全局日志体系，按时间、用户、req_id 建立复合索引，实现简单的排障溯源。
5. 内容安全：服务端实现 DFA + Trie 敏感词过滤与标签清洗；配置支持环境变量注入，配合 Docker 容器化部署，提升多环境发布一致性与可维护性。
